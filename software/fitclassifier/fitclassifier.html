Project Overview

Motivation

As an avid Pinterest scroller, I often discover unique fashion pieces that catch my eye, but many of these items lack proper identification or purchase information. This project was born from the frustration of finding beautiful clothing and jewelry without knowing how to find or buy them. FitClassifier serves as a "Shazam for clothes" - allowing users to take a photo of any fashion item and find that exact item for purchase.

Problem Statement

Traditional fashion discovery relies heavily on manual search and browsing, which is time-consuming and often unsuccessful. Users need a way to:

  • Instantly identify clothing items from images
  • Find similar products available for purchase
  • Discover fashion items without knowing brand names or descriptions
  • Access a comprehensive database of fashion products

Solution

FitClassifier combines computer vision and natural language processing to create an intelligent fashion recognition system that:

  • Detects clothing items using YOLOv8 object detection
  • Generates detailed descriptions using CLIP vision-language model
  • Performs semantic search using FAISS vector database
  • Provides real-time product recommendations

Technical Implementation

1. Object Detection (YOLOv8)

Custom-trained YOLOv8 model for fashion item detection. The model was trained on a diverse dataset of clothing items, optimized for speed and accuracy on consumer hardware.

2. Vision-Language Processing (CLIP)

OpenAI's CLIP model generates detailed textual descriptions of detected clothing items, enabling semantic understanding of fashion features.

3. Vector Search (FAISS)

Facebook AI Similarity Search (FAISS) enables fast similarity search across millions of product embeddings, providing real-time product recommendations.

4. Backend Infrastructure

Docker containerized Python backend deployed on Google Cloud Platform with Firestore for scalable data storage and retrieval.

Key Challenges

Product Search Calculation

The most challenging aspect was creating a FAISS index large and diverse enough to cover different types of clothing articles. Finding ways to embed unique fashion features in a manner that could be extracted and compared to new clothing inputs was crucial for system performance.

Model Size vs. Performance Trade-off

I chose a smaller YOLOv8 model for faster training on my laptop, but this came at the cost of accuracy. The model provides "mediocre performance" but serves as a solid foundation for future improvements.

Key Features

Image Upload

Users can upload photos of clothing items through an intuitive drag-and-drop interface or file selection.

Real-time Detection

YOLOv8 model instantly detects and localizes clothing items within uploaded images.

Semantic Description

CLIP model generates detailed, human-readable descriptions of detected fashion items.

Product Recommendations

FAISS vector search provides similar product recommendations with purchase links.

User Experience

FitClassifier is designed as a casual tool that people can use to look up clothes they're interested in, similar to how people Google things to learn more about them. Users can "FitClassifier" outfits they're interested in and find out where they come from.

Performance Analysis

Training Data Distribution

The model shows uneven performance across clothing categories due to imbalanced training data. Categories like "Bottoms" and "Shoes" perform well (85% accuracy), while "Outerwear" struggles (65% accuracy) due to insufficient training examples.

Training data distribution showing class imbalance

Detection Performance

The YOLOv8 model achieves "mediocre performance" overall, with the smaller model size chosen for faster training on consumer hardware. This trade-off between speed and accuracy is evident in the detection results.

Validation batch showing detection results
Normalized confusion matrix showing classification performance

Precision-Recall Analysis

The PR curve and F1 curve indicate areas for improvement, particularly in categories with limited training data. The model shows good precision for well-represented classes but struggles with underrepresented categories.

Future Improvements

To enhance performance, I would focus on:

  • Expanding the training dataset with more balanced class distribution
  • Implementing data augmentation techniques
  • Exploring larger model architectures for better accuracy
  • Fine-tuning hyperparameters for optimal performance
